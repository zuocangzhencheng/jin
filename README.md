# 主线选股监控台

一个面向 A 股散户的本地静态选股监控 App。它不是荐股工具，而是一个帮助使用者执行交易纪律的盯盘系统：先看主线，再看趋势，再看量能和资金，最后才讨论买点和卖点。

当前版本：`v1.4.9 模块折叠`

## 项目定位

这个 App 解决三个核心问题：

1. 为什么盯？
2. 什么条件才买？
3. 错了在哪里卖？

它把自选股、市场环境、主线强度、资金异动、集合竞价、K 线状态、买卖点缺口、黄金价格和复盘日志放在同一个监控台里，帮助使用者少做冲动交易，多做条件确认。

## 快速开始

不需要安装依赖。

直接打开：

```text
outputs/stock-watch-app/index.html
```

如果浏览器限制本地行情请求，可以用任意静态服务器打开。例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/outputs/stock-watch-app/
```

## 当前功能

### 1. 自选股监控

- 支持持仓和观察分组。
- 支持成本价、目标价、止损价。
- 支持输入股票代码或常见股票名称自动识别。
- 支持自选列表本地保存。
- 支持自动刷新，默认 60 秒一次。

### 2. 模块折叠

- 每个大模块都支持单独收起和展开。
- 顶部支持一键全部折叠、全部展开。
- 折叠状态会保存到本地，刷新页面后继续沿用上次布局。

### 3. 交易三问

每只自选股卡片会显示：

- 为什么盯：主线强度、评分、趋势、量能、资金雷达等原因。
- 什么条件买：严格买点是否满足，以及当前还差什么条件。
- 错了在哪里卖：止损、跌破关键均线、放量分歧、目标价分批等风控条件。

核心原则：

- 新闻不是买点。
- 异动不是买点。
- 买点必须同时看市场环境、主线、K 线、量能、资金和风控。

### 4. 严格买卖点

当前买点包括：

- 放量突破
- 缩量回踩
- 弱转强
- 确认买点

卖点和风控包括：

- 到止损
- 跌破 20 日线
- 跌破 10 日线且不是缩量调整
- 放量分歧
- 到目标价后分批

如果数据质量不足，系统会阻断买点，提示先核验数据。

自选卡片新增“买点六灯”：

- 趋势灯
- 量能灯
- 主线灯
- 环境灯
- 风控灯
- 形态灯

数据可信度是前置闸门，不计入六灯；数据不足时直接阻断确认买点。

### 5. 中军 / 龙头自动切换

App 内置非创业板、非科创板的全市场主线核心池，并按主线强度和资金共振自动排序。

主线分为两层：

- 主攻科技：半导体、AI 算力、CPO、HBM、先进封装、电子特气等优先方向。
- 轮动观察：有色、黄金、稀土、煤炭能源、风电设备等非主攻方向。

轮动观察不会和主攻科技同权重抢位置，只有板块强度、资金共振和核心股趋势同时确认时，才会升级为轮动确认。

每条主线会标注：

- 中军：看板块持续性和大资金态度。
- 龙头：看板块弹性和短线强度。
- 分支核心：看细分方向是否扩散。

当其他板块成为领涨方向时，模块会自动把对应板块的中军和龙头排到前面。自选卡片中也会标注“中军”“龙头”“分支核心”“领涨主线”。

默认展示全部已配置主线核心池，避免先进封装、电子特气、光通信/海缆、风电设备、有色金属、黄金、稀土等重要分支被短期最强主线挤出视野。

### 6. K 线状态

使用日 K 数据判断：

- 是否站上 MA5 / MA10 / MA20
- 均线是否多头排列
- 距离 20 日高点的位置
- 当前是多头、修复、反抽还是弱势

如果缺少 K 线数据，系统不会给确认买点。

### 7. 资金异动雷达

雷达会扫描非创业板、非科创板、非北交所的主板大市值股票，默认过滤市值小于 300 亿的股票。

重点识别：

- 主力资金流入
- 大成交额异动
- 涨跌幅异常
- 板块归属
- 主线映射

雷达股票可以一键加入观察或设为持仓。

### 8. 资金流向

市场环境下方会显示资金流向方向，直接展示：

- 流入方向
- 主力净流入
- 成交额
- 平均涨幅
- 主攻科技 / 轮动观察 / 轮动确认
- 代表异动个股

这个模块用于快速判断钱正在流向哪里，不替代买点确认。

资金流向下方带有时间轴：

- 9:35：开盘确认
- 10:30：上午持续
- 13:30：午后回流
- 14:30：尾盘确认
- 最新：最近一次雷达快照

时间轴会标注持续流入、方向切换、冲高回落、尾盘抢筹、资金分歧等状态，用来判断资金是否真的连续。

### 9. 主线聚合雷达

把资金异动股票按主线聚合，观察是否出现板块共振。

示例主线：

- AI 算力 / 服务器
- 光模块 / CPO
- HBM / 存储
- 半导体设备
- 半导体材料
- 先进封装
- 电子特气
- 面板 / 玻璃基板
- 光通信 / 海缆
- 风电设备
- 有色金属
- 贵金属 / 黄金
- 稀土永磁
- 煤炭能源

### 10. 集合竞价观察

在 9:15 到 9:25 期间记录自选股竞价状态，并区分：

- 可撤单阶段
- 不可撤单阶段
- 强竞价
- 弱竞价
- 高开过热
- 9:35 承接验证

### 11. 黄金价格

每分钟随主刷新加载黄金价格，用于观察黄金、美债、风险偏好和资源品方向的映射。

### 12. 自选同步 / 复盘日志

支持：

- 复制自选同步文本
- 下载 JSON 备份
- 导入 JSON 备份
- 手动写入复盘
- 自动记录强信号、买点、卖点、风控、数据异常

这个模块用于配合每天早上 8 点和晚上 8 点的简报/复盘。

## 数据源与可靠性

当前使用免费行情数据源，可能出现：

- 请求失败
- 跨域失败
- 行情延迟
- 字段缺失
- 节假日或非交易时间数据不更新

因此 App 内置了数据可信判断：

- 行情是否加载
- K 线是否加载
- 行情时间是否过旧
- K 线日期是否过旧
- 实时价和 K 线收盘价是否异常偏离

只要数据可信不足，系统会提示“等数据”或“需核验”，不会把信号当成买点。

## GitHub Pages 部署

上传到 GitHub 后，可以用 GitHub Pages 直接部署静态页面。

推荐步骤：

1. 创建 GitHub 仓库。
2. 上传当前项目。
3. 打开仓库的 `Settings`。
4. 找到 `Pages`。
5. Source 选择 `Deploy from a branch`。
6. Branch 选择 `main`，目录选择 `/root`。
7. 页面地址通常是：

```text
https://<你的用户名>.github.io/<仓库名>/
```

根目录的 `index.html` 会自动跳转到 App 页面。

## 迭代方式

建议以后按这个节奏迭代：

1. 先明确问题：比如买点太松、日志太乱、数据不准。
2. 改规则或界面。
3. 记录到 `CHANGELOG.md`。
4. 如果是新想法，先写进 `ROADMAP.md`。
5. 再发布到 GitHub。

## 风险提示

本项目只用于个人学习、复盘和交易纪律辅助，不构成投资建议。

任何买卖都应以使用者自己的判断、风险承受能力和合规账户权限为准。A 股行情波动大，免费数据源不保证实时性、完整性和准确性。
