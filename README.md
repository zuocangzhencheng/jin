# 主线选股监控台

一个面向 A 股散户的本地静态选股监控 App。它不是荐股工具，而是一个帮助使用者执行交易纪律的盯盘系统：先看主线，再看趋势，再看量能和资金，最后才讨论买点和卖点。

当前版本：`v1.3 交易复盘`

## 项目定位

这个 App 解决三个核心问题：

1. 为什么盯？
2. 什么条件才买？
3. 错了在哪里卖？

它把自选股、市场环境、主线强度、资金异动、集合竞价、K 线状态、买卖点缺口、黄金价格和复盘日志放在同一个监控台里，帮助使用者少做冲动交易，多做条件确认。

## 快速开始

不需要安装依赖。

直接打开：

```text
outputs/stock-watch-app/index.html
```

如果浏览器限制本地行情请求，可以用任意静态服务器打开。例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/outputs/stock-watch-app/
```

## 当前功能

### 1. 自选股监控

- 支持持仓和观察分组。
- 支持成本价、目标价、止损价。
- 支持输入股票代码或常见股票名称自动识别。
- 支持自选列表本地保存。
- 支持自动刷新，默认 60 秒一次。

### 2. 交易三问

每只自选股卡片会显示：

- 为什么盯：主线强度、评分、趋势、量能、资金雷达等原因。
- 什么条件买：严格买点是否满足，以及当前还差什么条件。
- 错了在哪里卖：止损、跌破关键均线、放量分歧、目标价分批等风控条件。

核心原则：

- 新闻不是买点。
- 异动不是买点。
- 买点必须同时看市场环境、主线、K 线、量能、资金和风控。

### 3. 严格买卖点

当前买点包括：

- 放量突破
- 缩量回踩
- 弱转强
- 确认买点

卖点和风控包括：

- 到止损
- 跌破 20 日线
- 跌破 10 日线且不是缩量调整
- 放量分歧
- 到目标价后分批

如果数据质量不足，系统会阻断买点，提示先核验数据。

### 4. K 线状态

使用日 K 数据判断：

- 是否站上 MA5 / MA10 / MA20
- 均线是否多头排列
- 距离 20 日高点的位置
- 当前是多头、修复、反抽还是弱势

如果缺少 K 线数据，系统不会给确认买点。

### 5. 资金异动雷达

雷达会扫描非创业板、非科创板、非北交所的主板大市值股票，默认过滤市值小于 300 亿的股票。

重点识别：

- 主力资金流入
- 大成交额异动
- 涨跌幅异常
- 板块归属
- 主线映射

雷达股票可以一键加入观察或设为持仓。

### 6. 主线聚合雷达

把资金异动股票按主线聚合，观察是否出现板块共振。

示例主线：

- AI 算力 / 服务器
- 光模块 / CPO
- HBM / 存储
- 半导体设备
- 半导体材料
- 先进封装
- 电子特气
- 面板 / 玻璃基板
- 光通信 / 海缆

### 7. 集合竞价观察

在 9:15 到 9:25 期间记录自选股竞价状态，并区分：

- 可撤单阶段
- 不可撤单阶段
- 强竞价
- 弱竞价
- 高开过热
- 9:35 承接验证

### 8. 黄金价格

每分钟随主刷新加载黄金价格，用于观察黄金、美债、风险偏好和资源品方向的映射。

### 9. 自选同步 / 复盘日志

支持：

- 复制自选同步文本
- 下载 JSON 备份
- 导入 JSON 备份
- 手动写入复盘
- 自动记录强信号、买点、卖点、风控、数据异常

这个模块用于配合每天早上 8 点和晚上 8 点的简报/复盘。

## 数据源与可靠性

当前使用免费行情数据源，可能出现：

- 请求失败
- 跨域失败
- 行情延迟
- 字段缺失
- 节假日或非交易时间数据不更新

因此 App 内置了数据可信判断：

- 行情是否加载
- K 线是否加载
- 行情时间是否过旧
- K 线日期是否过旧
- 实时价和 K 线收盘价是否异常偏离

只要数据可信不足，系统会提示“等数据”或“需核验”，不会把信号当成买点。

## GitHub Pages 部署

上传到 GitHub 后，可以用 GitHub Pages 直接部署静态页面。

推荐步骤：

1. 创建 GitHub 仓库。
2. 上传当前项目。
3. 打开仓库的 `Settings`。
4. 找到 `Pages`。
5. Source 选择 `Deploy from a branch`。
6. Branch 选择 `main`，目录选择 `/root`。
7. 页面地址通常是：

```text
https://<你的用户名>.github.io/<仓库名>/outputs/stock-watch-app/
```

## 迭代方式

建议以后按这个节奏迭代：

1. 先明确问题：比如买点太松、日志太乱、数据不准。
2. 改规则或界面。
3. 记录到 `CHANGELOG.md`。
4. 如果是新想法，先写进 `ROADMAP.md`。
5. 再发布到 GitHub。

## 风险提示

本项目只用于个人学习、复盘和交易纪律辅助，不构成投资建议。

任何买卖都应以使用者自己的判断、风险承受能力和合规账户权限为准。A 股行情波动大，免费数据源不保证实时性、完整性和准确性。
